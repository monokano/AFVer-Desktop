Printing and Exporting

The list you are viewing can be kept as a print, PDF, or CSV. If you output it with the versions shown, you get a record that tells you the contents and their versions without opening any files. Print the contents of a CD-R or DVD-R and keep it together with the disc, and you can check what is inside without mounting it.

Printing

  1. Expand the list to the state you want to print (use Expand All if needed).
  2. Choose "Print…" (⌘P) from the "File" menu, or click the Print button in the toolbar.
  3. The standard macOS print dialog opens. Check the preview, then print or save as PDF.
Note What gets printed is "the state currently expanded on screen." The contents of collapsed folders are not printed. To show every level, use Expand All first.

What is printed

  • Header: the icon, name, and total size of the open folder (or volume). The print date and time appears at the far right.
  • List: the same columns as on screen. Folders show a mark indicating their expanded state (∨ / >). Icons are printed as well. When the Version column is shown, the color dots at the start of each row are printed too.
  • Footer: the page number.

Page orientation

The default is A4 portrait. When the Version column is shown (the switch is on), the orientation automatically becomes landscape, because the extra column does not fit well across the page. You can also change the orientation manually in the print dialog.

About the total size When printing the root of a volume, the total used space of the entire disk is shown immediately. For an ordinary folder, the contents are totaled, but if this does not finish within 3 seconds the size is not shown (so that no incomplete value is displayed).

Export as CSV

You can save the contents of the list as a CSV file that opens in spreadsheet apps. This is handy when you want to keep a searchable catalog.

  1. Expand the list to the state you want to export.
  2. Choose "Export as CSV…" (⌘E) from the "File" menu.
  3. Choose a destination and save.

As with printing, what gets exported is "the state currently expanded on screen." The Name column uses indentation so that the hierarchy is clear.

Tip The exported CSV opens directly in Excel (it is set up so the text does not become garbled).
